# !wrongsong

<PlatformBadges supported={[ 'Twitch' ]} />

## Overview

The `!wrongsong` command allows viewers to remove the **last song they added** to the StreamElements media request queue. This is useful if you accidentally requested the wrong song or simply changed your mind.

## Usage

To remove your last song request, simply type the command in chat:

```streamelements
!wrongsong
```

## Examples

<ChatExample messages={[
  { persona: 'user', message: '!wrongsong' },
  { persona: 'bot', message: '@ViewerName Removed your last song from the queue: Bohemian Rhapsody by Queen' } // Output varies
]} />

<ChatExample messages={[

## Parameters

The `!wrongsong` command does not require any parameters.

## Configuration

*   Requires the **Media Request** module to be enabled.
*   The command affects only the song requested by the user issuing the command.
*   Streamers can customize the response messages in the StreamElements dashboard under `Chatbot` -> `Chat Commands` -> `Default Commands`.

## Related Commands

*   [`!songrequest`](songrequest.mdx): Used to add a song to the request queue.
*   [`!when`](when.mdx): Checks the status of your last requested song.
