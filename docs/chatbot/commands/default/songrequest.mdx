# !songrequest

<PlatformBadges supported={[ 'Twitch', 'YouTube', 'Trovo', 'Kick' ]} />

## Overview

The `!songrequest` command (aliased as `!sr`) allows viewers to add songs or videos to the streamer's media request queue. This command primarily integrates with YouTube, enabling users to request media using search terms or direct URLs.

## Usage

To request media:

```streamelements
!songrequest <search_term_or_url>
```

Or using the alias:

```streamelements
!sr <search_term_or_url>
```

## Parameters

*   `<search_term_or_url>` (Required): The media to request. This can be:
    *   A YouTube video URL.
    *   A YouTube playlist URL (if enabled in settings).
    *   A search term (the bot will typically pick the first result).

## Configuration

*   Requires the **Media Request** module to be enabled and configured (`Chatbot` -> `Modules`).
*   Streamers can configure various settings in the Media Request module:
    *   Enable/disable the feature.
    *   Link a Spotify account (optional, for different functionality).
    *   Set request limits (per user, per stream).
    *   Define user level restrictions for requesting.
    *   Set maximum song/video duration.
    *   Enable/disable playlist requests.
    *   Set up a song blacklist (by keyword or specific URL).
    *   Configure point costs or subscriber-only access.
