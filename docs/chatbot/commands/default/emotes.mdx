# !emotes

<PlatformBadges supported={[ 'Twitch', 'YouTube', 'Trovo', 'Kick' ]} />

## Overview

The `!emotes` command serves two primary functions:

1.  **List Platform Emotes:** When followed by a platform name (`twitch`, `bttv`, `ffz`, `7tv`), it lists the active emotes from that specific provider directly in chat.
2.  **Reload Emote Cache (Moderator+):** Using the `reload` keyword, moderators can refresh the bot's internal cache of emotes for **all** platforms (Twitch, BTTV, FFZ, 7TV) simultaneously.

## Usage

**To list active emotes for a specific platform:**

```streamelements
!emotes <platform>
```
*   `<platform>`: Required. The platform whose emotes you want to list. Valid options: `twitch`, `bttv`, `ffz`, `7tv`.

**To reload the emote cache:**

```streamelements
!emotes reload
```
*   This refreshes the bot's internal cache of available emotes for **all** platforms (Twitch, BTTV, FFZ, 7TV).
*   It is useful if new emotes have been added or changed and aren't showing up correctly.
<Admonition type="important" title="Permissions">
  Using `!emotes reload` requires **Moderator** permission level or higher by default. This can be adjusted in the StreamElements Dashboard under `Chatbot` -> `Chat Commands` -> `Default Commands`.
</Admonition>

## Configuration

*   The `reload` function will refresh caches for Twitch and any enabled third-party integrations.
*   The bot needs appropriate permissions (usually just being present in chat) to fetch Twitch emotes.
