# !bet

<PlatformBadges supported={[ 'Twitch' ]} />

## Overview

The `!bet` command allows viewers to use their loyalty points to bet on the outcome of a contest or event defined by the streamer. This command is used **while a bet is active**, which is initiated through the StreamElements dashboard or via chatbot commands like [`!contest`](contest.mdx).

## Usage

To place a bet while a betting contest is open, viewers use the following syntax:

```streamelements
!bet <option> <points>
```

## Parameters

*   `<option>` (Required): The specific betting option the viewer wants to choose (defined by the streamer when starting the bet).
*   `<points>` (Required): The number of loyalty points to wager. This can be a specific number, a percentage (`50%`), or use `k` for thousands (`5k`), `m` for millions (`1m`).

## Configuration

*   Betting contests are managed via the **Betting module** in the StreamElements dashboard or through commands like [`!contest`](contest.mdx). Streamers define the options, duration, and other settings there.
*   Viewers need sufficient loyalty points to place a bet.

## Related Commands

*   [`!contest`](contest.mdx): Starts a betting contest (requires Moderator permissions).
*   [`!points`](points.mdx): Checks a user's current loyalty point balance.
